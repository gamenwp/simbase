User Datagram Protocol

May 10, 2022 Roel van der Meer

User Datagram Protocol (UDP) a connectionless transport-layer protocol. It provides an unreliable datagram connection between applications. Data is transmitted link by link and there is no end-to-end connection. The protocol assumes that the Internet Protocol (IP) is used as the underlying protocol to route packets of data (datagram) through the network. UDP does not set up a connection between the source and destination. It just pushes packets of data out on the network and accepts incoming datagrams at the receiver. UDP does not guarantee delivery or the correct order of arrival. UDP only provides the possibility to use a checksum to check the integrity of the data at the receiver. 

UDP is used by applications where speed is more important than reliable delivery of the data. An example of applications which makes use of UDP is instant messaging. UDP port is a channel in which the communication between an application on a computer and UDP pass through. The combination of IP address and port number is called a socket. An application that is offering service waits for messages to come in on a specific port dedicated to that service. This makes it possible to have more than one UDP services running on a computer.
